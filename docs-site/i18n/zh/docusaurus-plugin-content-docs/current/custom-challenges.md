# 自定义反垃圾邮件挑战

Bitsocial 并不假设存在一种通用的垃圾邮件防御机制。每个社区都可以决定有效的帖子或回复需要什么，并且该挑战可以是轻量级的，也可以是固执己见的，取决于操作员的意愿。

## 挑战可以是什么

- 验证码
- 帐户年龄或信誉检查
- 短信验证
- 小额支付
- 代币或NFT要求
- 基于IP的规则
- 白名单
- 可以用代码表达的任何其他策略

## 为什么这很重要

垃圾邮件抵抗保留在社区本地，而不是外包给协议范围内的审核机构。托管社区的节点可以点对点地传达挑战，这意味着社区可以调整其防御措施，而无需强制整个网络采用一项策略。

## 这会带来什么变化

在集中式平台上，反垃圾邮件策略通常会变成产品策略。发生这种情况时，由一家公司决定什么算作有效帐户、有效帖子或可接受的覆盖范围。

Bitsocial 打破了这种耦合：

- 协议保持中立
- 社区选择挑战
- 应用程序决定如何呈现或抽象该挑战

这更适合用于托管多种社交产品而不是全球默认产品的网络。
