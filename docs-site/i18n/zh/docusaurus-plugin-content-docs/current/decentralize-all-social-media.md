# 去中心化所有社交媒体

这是 Bitsocial 总体规划所描述的最终状态：不是一个应用程序，而是一个可以与社交媒体的每个主要类别相媲美的 Bitsocial 应用程序生态系统。

## 该模型

在拟议的网络成熟状态中：

- 多个公共 RPC 提供商竞争托管配置文件节点
- 用户仍然可以自行托管
- 提要算法成为可选服务，而不是强制性平台法
- 货币化并不依赖于银行愿意让平台保持活力
- 应用程序在产品质量、发现和界面设计上进行竞争

## 成功会是什么样子

Bitsocial应用程序应该能够挑战跨格式的平台，而不仅仅是在一个利基市场：

- 文字和讨论
- 图片板
- 论坛
- 创作者社区
- 短视频和更广泛的媒体产品

我们的目标不是强迫每个人都使用一种通用界面。它是为了让许多应用程序能够访问相同的底层社区、身份和网络。

## 为什么 RPC 层在这里很重要

一旦公共 RPC 市场存在，永远在线的社区和配置文件就会变得更加实用。如果没有这一层，自托管对于许多用户来说仍然是太高的摩擦，尤其是在移动设备上。

## 为什么网络层在这里很重要

如果内容货币化可以被普通银行或平台压力切断，那么生态系统就会保持脆弱。 Bitsocial Network 被提议作为使长期竞争更加可信的经济层。

## 核心赌注

最大的赌注是社交媒体可以分解为可替换的层：

- 协议
- 社区
- 身份
- 托管
- 发现
- 货币化
- 应用

一旦这些层可替换，任何一家公司都不必拥有整个社交图谱才能使产品发挥作用。
