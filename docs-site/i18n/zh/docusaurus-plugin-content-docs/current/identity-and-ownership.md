# 身份和社区所有权

Bitsocial 中的个人资料和社区由密钥对控制，而不是由可撤销的平台帐户控制。

这改变了所有权和服务提供商之间的关系。您可以将正常运行时间、托管或便利性委托给其他服务，而无需让该服务对身份或社区本身进行永久托管。

## 为什么密钥在这里很重要

托管平台帐户最终是租用的。只要平台继续遵守它，它就存在。

密钥控制的身份是不同的：

- 私钥是权限的来源
- 托管可以被替换
- 应用程序选择可以改变
- 所有权不需要随基础设施移动

## 委托意味着什么

委派应该意味着提高可用性，而不是放弃控制权。

例如，公共 RPC 或托管服务可以使节点保持在线，但该服务不应该需要永久夺取配置文件或社区的权力。帮助很有用。托管应该保持可选。

## 更广泛的观点

这使得 Bitsocial 身份感觉更接近于钱包控制的资产，而不是从公司租用的用户名。服务层可以换出。所有者关系应该在交换后继续存在。
