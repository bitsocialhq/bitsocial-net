# 本地审核，而非全球禁令

Bitsocial 并不是没有节制的。这是没有协议级超级管理员的调节。

社区所有者仍然为自己的空间制定规则。应用程序仍然决定它们索引、排名或突出显示的内容。 Bitsocial 不包括可以没收社区或从网络本身删除身份的全球权威机构。

## 审核如何运作

- 社区所有者可以审核该社区
- 应用可以选择索引或显示的内容
- 用户可以移动到另一个社区或另一个应用

如果一个应用过滤了您，另一个应用仍然可以选择向您显示。如果一个社区拒绝您，另一个社区可以接受您，或者您也可以创建自己的社区。

## Bitsocial 避免什么

Bitsocial 避免了由一家公司控制的常见平台瓶颈：

- 社交图谱
- 审核数据库
- 发现界面
- 帐户命名空间

这个中心点是将产品政策决策转变为全网络禁令的原因。

## 实际权衡

本地审核并不意味着每个应用程序看起来都一样。这意味着适度的后果仍然是有限的：

- 社区规则适用于该社区内部
- 应用程序规则适用于该应用程序
- 不会自动成为协议法

这就是“无全球禁令”试图保留的内容。
